## Directory Purposes

**`src/`:**
- Purpose: All application source code.
- Contains: NestJS modules, controllers, services, DTOs, utilities.
- Key files: `src/main.ts`, `src/app.module.ts`, `src/db/schema.ts`

**`src/account/`:**
- Purpose: App Store–compliant account deletion and GDPR data export.
- Contains: `account.controller.ts`, `account.service.ts`, `account.module.ts`, `account.service.spec.ts`

**`src/categories/`:**
- Purpose: User expense category management with lazy default initialization and Redis caching.
- Contains: `categories.controller.ts`, `categories.service.ts`, `categories.module.ts`, `categories.service.spec.ts`

**`src/db/`:**
- Purpose: Database connectivity and schema definition.
- Contains: `client.ts` (raw client), `database.module.ts` (NestJS DI module), `db-token.ts` (injection symbol), `schema.ts` (Drizzle table definitions)

**`src/input/`:**
- Purpose: AI-powered expense parsing from raw Vietnamese text and receipt images.
- Contains: `input.controller.ts`, `input.service.ts`, `input.module.ts`, `rate-limit.guard.ts`, `dto/`, `*.spec.ts`

**`src/insights/`:**
- Purpose: Monthly spending analytics, category breakdowns, and daily expense aggregations.
- Contains: `insights.controller.ts`, `insights.service.ts`, `insights.module.ts`, `insights.service.spec.ts`

**`src/lib/`:**
- Purpose: Cross-cutting utilities and external service configuration.
- Contains: `auth.ts`, `redis.ts`, `openrouter.ts`, `model-config.ts`, `prompts.ts`, `image-resize.ts`, `request-context.ts`, `request-context.interceptor.ts`, `timeout.interceptor.ts`, `shutdown.service.ts`, `merchant-table.ts`, `date-utils.ts`, plus `*.spec.ts` test files.

**`src/transactions/`:**
- Purpose: Core expense CRUD with pagination and budget alert side effects.
- Contains: `transactions.controller.ts`, `transactions.service.ts`, `transactions.module.ts`, `dto/`, `*.spec.ts`

**`test/`:**
- Purpose: Shared test infrastructure.
- Contains: `helpers/setup.ts` (in-memory SQLite bootstrap for Jest)
- Note: Unit and integration tests are co-located inside `src/**/*.spec.ts`; this directory does not contain e2e tests yet.

**`drizzle/`:**
- Purpose: Database migration artifacts generated by Drizzle Kit.
- Contains: SQL migration files and `meta/` directory.
- Generated: Yes (via `drizzle-kit generate`)
- Committed: Yes (migration history is committed)

**`docs/`:**
- Purpose: Human-readable project documentation.
- Contains: Integration guides, OAuth setup instructions, architecture specs.

**`.planning/`:**
- Purpose: GSD planning artifacts (milestones, phases, quick fixes, research).
- Contains: Markdown files consumed by GSD tooling.

## Key File Locations

**Entry Points:**
- `src/main.ts`: Application bootstrap, Swagger, CORS, helmet, global configuration, Vercel handler export.
- `src/instrument.ts`: Sentry initialization; must be imported before all other modules in `main.ts`.

**Configuration:**
- `package.json`: Project metadata, dependencies, npm scripts, Jest configuration.
- `tsconfig.json`: TypeScript compiler options (`nodenext` module resolution, decorators enabled, strict mode).
- `nest-cli.json`: NestJS schematic collection, source root, compiler options.
- `drizzle.config.ts`: Drizzle Kit schema path, output directory, Turso dialect credentials.
- `vercel.json`: Vercel builder (`@vercel/node`), catch-all route, `NODE_OPTIONS` env.
- `eslint.config.mjs`: ESLint flat config with Prettier integration.
- `.prettierrc`: Code formatting rules.

**Core Logic:**
- `src/db/schema.ts`: Single source of truth for all database tables (`user`, `session`, `account`, `transactions`, `categories`, `pushTokens`, `notificationPreferences`).
- `src/app.module.ts`: Root module wiring all feature modules, global providers (Sentry filter, request context interceptor), and configuration validation.
- `src/lib/auth.ts`: Better Auth instance configuration (OAuth providers, plugins, database adapter).

**Testing:**
- `test/helpers/setup.ts`: Jest setup file creating an in-memory `better-sqlite3` database and running Drizzle migrations before tests.
- `src/**/*.spec.ts`: Co-located unit tests (e.g., `src/transactions/transactions.service.spec.ts`, `src/input/input.service.spec.ts`).

## Naming Conventions

**Files:**
- Feature modules use `[feature].[type].ts` (e.g., `transactions.controller.ts`, `transactions.service.ts`, `transactions.module.ts`).
- DTOs live in `[feature]/dto/[action]-[entity].dto.ts` (e.g., `create-transaction.dto.ts`).
- Test files are co-located as `[feature].[type].spec.ts`.
- Utilities in `src/lib/` use `[purpose].ts` (e.g., `redis.ts`, `openrouter.ts`).

**Directories:**
- Feature modules are named with plural nouns: `transactions`, `categories`, `insights`.
- DTO subdirectories are named `dto/` inside their parent feature module.

## Where to Add New Code

**New Feature Module:**
- Create a new directory under `src/[feature]/`.
- Add `[feature].module.ts`, `[feature].controller.ts`, `[feature].service.ts`.
- Register the module in `src/app.module.ts` imports array.

**New API Endpoint:**
- Add route handler to the relevant `src/[feature]/[feature].controller.ts`.
- Implement business logic in `src/[feature]/[feature].service.ts`.
- Define request/response DTOs in `src/[feature]/dto/`.
- Add Swagger decorators (`@ApiOperation`, `@ApiResponse`, `@ApiBody`) for documentation.

**New Database Table:**
- Add table definition to `src/db/schema.ts` using `sqliteTable` from `drizzle-orm/sqlite-core`.
- Export inferred types (`export type NewTable = typeof table.$inferInsert`).
- Generate migration: `npx drizzle-kit generate`.
- Apply migration: `npx drizzle-kit migrate`.

**New External Integration:**
- Add client factory or configuration to `src/lib/` (follow existing patterns like `src/lib/openrouter.ts` or `src/lib/redis.ts`).
- Inject into services via NestJS DI or import directly for stateless clients.

**New Cross-Cutting Utility:**
- Add to `src/lib/[utility].ts`.
- If it requires NestJS lifecycle (interceptor, guard, filter), register it in `src/app.module.ts` providers array (or as `APP_INTERCEPTOR` / `APP_FILTER` provider).

**New Unit Test:**
- Co-locate as `src/[feature]/[feature].service.spec.ts` or `src/lib/[utility].spec.ts`.
- Ensure `test/helpers/setup.ts` is referenced in Jest config (`setupFilesAfterEnv`) if the test needs the database.

- Purpose: Compiled JavaScript output from `nest build`.
- Generated: Yes

- Purpose: npm dependencies.
- Generated: Yes
- Committed: No (gitignored)

**`drizzle/meta/`:**
- Purpose: Drizzle Kit migration journal and snapshots.
- Generated: Yes

**`.planning/`:**
- Purpose: GSD planning artifacts consumed by project orchestration tools.
- Generated: Partially (by GSD agents)
