## File Responsibilities

### Root Files

| File | Purpose |
|------|---------|
| `src/main.ts` | Application entry point. Exports `bootstrap()` (creates NestJS Express app with helmet, CORS, ValidationPipe) and default `handler()` (Vercel serverless pattern with cached app instance). Also supports local `listen()` mode for development. |
| `src/app.module.ts` | Root NestJS module. Imports `ConfigModule` (global env), `LoggerModule` (pino with request header redaction), `AuthModule` (Better Auth bridge), and all 5 feature modules. Registers `HealthController`. |
| `src/health.controller.ts` | Simple health check at `GET /api/health`. Returns `{ status: "ok", timestamp }`. Marked `@AllowAnonymous()` to bypass auth. |
| `package.json` | Project manifest. Node >=20.x. Scripts: `build`, `start:dev`, `start:prod`, `format`, `lint`, `test`, `test:e2e`. 22 runtime deps, 22 dev deps. |
| `tsconfig.json` | TypeScript strict mode, target ES2023, module NodeNext, experimental decorators (required by NestJS), emitDecoratorMetadata. Output to `./dist`. |
| `vercel.json` | Routes all requests to `dist/main.js` via `@vercel/node` builder. |
| `nest-cli.json` | NestJS CLI: source root is `src`, delete output dir on build. |
| `eslint.config.mjs` | ESLint v9 flat config with TypeScript ESLint (recommendedTypeChecked), Prettier plugin, Node+Jest globals. Relaxes `no-explicit-any`. |
| `drizzle.config.ts` | Drizzle Kit: schema at `src/db/schema.ts`, output to `./drizzle`, Turso dialect with env-based credentials. |
| `.env.example` | Template showing all required env vars: PORT, NODE_ENV, BETTER_AUTH_SECRET, BETTER_AUTH_URL, GitHub OAuth, Apple OAuth, Turso DB, OpenRouter API key, Upstash Redis, FRONTEND_URL. |

### Database Layer (`src/db/`)

| File | Purpose |
|------|---------|
| `src/db/client.ts` | Creates `@libsql/client` with Turso credentials from env vars (`TURSO_CONNECTION_URL`, `TURSO_AUTH_TOKEN`, `TURSO_SYNC_URL`) and wraps it with `drizzle()`. Falls back to local `file:local.db` if no URL provided. Exports `db` singleton. |
| `src/db/schema.ts` | Defines all 6 tables using `drizzle-orm/sqlite-core`: `user`, `session`, `account`, `verification` (Better Auth tables) + `transactions`, `categories` (app tables). Includes indices, unique constraints, foreign keys with cascading deletes. Exports `Transaction`, `NewTransaction`, `Category`, `NewCategory` TypeScript types. |

### Feature Modules

#### Account Module (`src/account/`)

| File | Purpose |
|------|---------|
| `src/account/account.module.ts` | Registers `AccountController` and `AccountService` in NestJS DI. |
| `src/account/account.controller.ts` | `DELETE /api/account` — deletes the authenticated user's account and all data. `GET /api/account/export` — exports all user transactions and categories as JSON. |
| `src/account/account.service.ts` | `deleteAccount()`: Runs a Drizzle transaction to cascade-delete rows from `transactions`, `categories`, `session`, `account`, and `user` tables (defensive deletion even with FK cascades). `exportData()`: Queries all user transactions and categories, returns JSON with export timestamp. |

#### Categories Module (`src/categories/`)

| File | Purpose |
|------|---------|
| `src/categories/categories.module.ts` | Registers `CategoriesController` and `CategoriesService` in NestJS DI. |
| `src/categories/categories.controller.ts` | `GET /api/categories` — lists all categories for the authenticated user. |
| `src/categories/categories.service.ts` | `list()`: Fetches user categories from DB. If none exist (new user), lazy-initializes 8 default Vietnamese categories (Ăn uống, Di chuyển, Mua sắm, Giải trí, Hóa đơn, Sức khỏe, Giáo dục, Khác) with `nanoid()` IDs. Handles concurrent initialization race condition by catching UNIQUE constraint violations and re-fetching. |

#### Transactions Module (`src/transactions/`)

| File | Purpose |
|------|---------|
| `src/transactions/transactions.module.ts` | Registers `TransactionsController` and `TransactionsService` in NestJS DI. |
| `src/transactions/transactions.controller.ts` | `GET /api/transactions?month=YYYY-MM` — list by month. `POST /api/transactions` — create new transaction. `PATCH /api/transactions/:id` — update transaction. `DELETE /api/transactions/:id` — delete transaction. All scoped to authenticated user. |
| `src/transactions/transactions.service.ts` | `listByMonth()`: Filters by `userId` + month prefix (LIKE query on `created_at` ISO string). `create()`: Inserts with negated amount (`-Math.abs`), `nanoid()` ID, ISO timestamps. `update()`: Partial update with amount negation if provided. `delete()`: Returns deleted row, throws `NotFoundException` if not found. |
| `src/transactions/dto/create-transaction.dto.ts` | Validates required fields: `amount` (positive int, min 1), `merchant` (max 255 chars), `category` (max 100 chars), `source` (enum: text/voice/image/sms/manual), `note` (optional, max 1000 chars). |
| `src/transactions/dto/update-transaction.dto.ts` | All fields optional: `amount`, `merchant`, `category`, `note` with same constraints. |

#### Input Module (`src/input/`)

| File | Purpose |
|------|---------|
| `src/input/input.module.ts` | Registers `InputController` and `InputService` in NestJS DI. |
| `src/input/input.controller.ts` | `POST /api/input/text` — parse natural language expense. `POST /api/input/image` — parse receipt image. Both protected by `@UseGuards(RateLimitGuard)` in addition to auth. |
| `src/input/input.service.ts` | `parseText()`: Two-tier parsing: checks `MERCHANT_CATEGORY_MAP` first, falls back to OpenRouter LLM (`qwen/qwen3-8b`). `parseImage()`: Sends base64 image to `gpt-4o-mini` via OpenRouter. Both use Vietnamese prompts, extract JSON from LLM response, fall back gracefully on errors. Includes local helpers for merchant extraction and VND amount regex parsing. |
| `src/input/rate-limit.guard.ts` | `RateLimitGuard` implements `CanActivate`. Uses `@upstash/ratelimit` sliding window (20 req/hour) per authenticated user ID (from `request.user.id`), falling back to IP-based limiting if no user context. Returns 429 on limit exceeded. |
| `src/input/dto/text-input.dto.ts` | `TextInputDto`: `message` string, max 500 chars. |
| `src/input/dto/image-input.dto.ts` | `ImageInputDto`: `image` base64 string, max ~15MB (15,000,000 chars). |

| File | Purpose |

## Naming Conventions

### Files

- **Module definition files:** `<name>.module.ts` (e.g., `transactions.module.ts`, `account.module.ts`)
- **Controller files:** `<name>.controller.ts` (e.g., `transactions.controller.ts`)
- **Service files:** `<name>.service.ts` (e.g., `transactions.service.ts`)
- **DTO files:** `create-<name>.dto.ts` / `update-<name>.dto.ts` inside a `dto/` subdirectory (e.g., `src/transactions/dto/create-transaction.dto.ts`)
- **Guard files:** `rate-limit.guard.ts` — snake-case with `.guard` suffix
- **Library/utility files:** descriptive kebab-case (e.g., `merchant-table.ts`, `openrouter.ts`, `prompts.ts`)
- **Schema/DB files:** `schema.ts`, `client.ts`
- **Config files:** `.config.*` or `.json` at root (e.g., `drizzle.config.ts`, `nest-cli.json`, `vercel.json`)

### Classes

- **Controllers:** PascalCase, suffixed with `Controller` (e.g., `HealthController`, `TransactionsController`)
- **Services:** PascalCase, suffixed with `Service` (e.g., `TransactionsService`, `InputService`)
- **Modules:** PascalCase, suffixed with `Module` (e.g., `TransactionsModule`, `AuthModule`)
- **DTOs:** PascalCase, suffixed with `Dto` (e.g., `CreateTransactionDto`, `UpdateTransactionDto`)
- **Guards:** PascalCase, suffixed with `Guard` (e.g., `RateLimitGuard`)

### Directories

- **Feature modules:** lowercase, plural where applicable (e.g., `transactions/`, `categories/`, `insights/`)
- **Infrastructure:** short lowercase (e.g., `db/`, `lib/`)
- **DTO subdirectory:** always `dto/` inside the feature directory
- **No barrel files (index.ts) are used** — imports use direct file paths.

### Functions / Methods

- **Public methods:** camelCase, descriptive (e.g., `listByMonth()`, `getMonthlyInsights()`, `deleteAccount()`)
- **Private helpers:** camelCase, prefixed with the action (e.g., `lookupMerchant()`, `parseAmount()`, `extractMerchant()`)
- **Singleton getters:** prefixed with `get` (e.g., `getOpenAIClient()`, `getRedisClient()`, `getRatelimitClient()`)

### Variables

- **Database tables (Drizzle):** lowercase plural (e.g., `transactions`, `categories`, `user`, `session`)
- **Singletons:** lowercase, module scope (e.g., `db`, `auth`, `redisClient`, `ratelimitClient`)
- **Constants:** UPPER_SNAKE_CASE for configuration values (e.g., `DEFAULT_CATEGORIES`, `MERCHANT_CATEGORY_MAP`, `MAX_MESSAGE_LENGTH`, `SYSTEM_PROMPT`)

### API Routes

- **Base prefix:** `/api/` for all routes
- **Resource naming:** lowercase plural nouns (e.g., `/api/transactions`, `/api/categories`, `/api/insights`)
- **Action naming:** descriptive verbs (e.g., `/api/input/text`, `/api/input/image`, `/api/account/export`)
- **Auth routes** are auto-generated by Better Auth under `/api/auth/*`

## Where to Add New Code

### New Feature Module

**Create a new directory** `src/<feature>/` with:
- `<feature>.module.ts` — NestJS module registration
- `<feature>.controller.ts` — Route handlers with `@Controller('api/<feature>')`
- `<feature>.service.ts` — Business logic with `@Injectable()`
- `dto/` subdirectory if the feature accepts request bodies

**Register the module** in `src/app.module.ts` imports array.

**Tests:** Add `src/<feature>/<feature>.service.spec.ts` for unit tests (Jest pattern: `.*\\.spec\\.ts$` per `package.json:85`). E2E tests go in `test/` directory.

### New DTO (validation class)

Create in `<feature>/dto/<name>.dto.ts`. Use `class-validator` decorators. If the DTO is shared between features, place it in a shared `src/dto/` directory (which does not yet exist).

### New Shared Utility

Place in `src/lib/<name>.ts`. Keep files small and single-purpose. Use lazy initialization pattern for external service clients (see `src/lib/redis.ts` or `src/lib/openrouter.ts`).

### New Database Table

Add the table definition in `src/db/schema.ts`. Follow the existing pattern: `drizzle-orm/sqlite-core` imports, column definitions with appropriate types/constraints, indices, foreign keys with cascade deletes. Export `$inferSelect` and `$inferInsert` types. Run `npx drizzle-kit generate` to create migrations.

### New Guard

Create in the feature module's directory as `<name>.guard.ts`, implementing `CanActivate`. Apply with `@UseGuards()` on controllers or individual routes.
