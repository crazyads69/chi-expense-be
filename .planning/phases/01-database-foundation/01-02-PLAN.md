Output: Updated client.ts importing from @libsql/client/http, synced .env.example documentation.
</objective>

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Spoofing | client.ts → Turso | mitigate | TLS enforced by @libsql/client/http for remote URLs (`libsql://` → HTTPS). Auth token sent as bearer token. Connection URL validated at client creation. |
| T-02-02 | Information Disclosure | client.ts | mitigate | authToken read from `process.env.TURSO_AUTH_TOKEN` — never hardcoded. .env.example contains placeholder values only. .env is in .gitignore. |
| T-02-03 | Denial of Service | client.ts | accept | HTTP client uses connection-per-request model (stateless). Turso free tier supports 500 concurrent connections — sufficient for solo dev / <1000 users. No connection pooling needed at this scale. |
