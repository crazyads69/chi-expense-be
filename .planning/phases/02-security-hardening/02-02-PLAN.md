```typescript
  private sanitizeUserMessage(message: string): string {
    // Strip characters that could be used for prompt injection:
    // - JSON/control characters: { } [ ] < > \x00-\x1F
    // - Backticks: ` (used to escape markdown/code blocks)
    // - Triple quotes: """ (Python-style string delimiters)
    // Note: We preserve Vietnamese diacritics and normal punctuation.
    return message
      .replace(/[{}[\]<>\x00-\x1F`]/g, '')
      .replace(/"""/g, '"')
      .trim();
  }
```

Replace the template to wrap the message in delimiters:

```typescript
export function USER_PROMPT_TEMPLATE(message: string): string {
  return `Phân tích tin nhắn sau và trả về JSON chi tiêu:

<<<USER_MESSAGE>>>
${message}
<<<END_USER_MESSAGE>>>

Ví dụ:
- "cà phê 35k" → {"amount": 35000, "merchant": "Cà phê", "category": "Ăn uống"}
- "Xăng 50k" → {"amount": 50000, "merchant": "Xăng dầu", "category": "Di chuyển"}
- "Mua sách 200k" → {"amount": 200000, "merchant": "Nhà sách", "category": "Giáo dục"}`;
}
```

**Key points:**
- Sanitization strips JSON brackets, control chars, backticks — prevents "ignore previous instructions" attacks
- Delimiter markers (`<<<USER_MESSAGE>>>`) help the LLM distinguish user content from instructions even if sanitization misses something
- The merchant lookup path (lines 74-79) does NOT need sanitization — it uses raw string matching, not LLM
- `parseAmount` and `extractMerchant` already have basic length checks

```typescript
import { IsString, IsNotEmpty, MaxLength, Matches } from 'class-validator';

}
```
