<task type="auto">
  <name>Task 1: Create GitHub Actions CI workflow</name>
  <files>.github/workflows/ci.yml</files>
  <action>
    Create `.github/workflows/ci.yml` with the following content:

    ```yaml
    name: CI

    on:
      pull_request:
        branches: [main, develop]
      push:
        branches: [main, develop]

    jobs:
      lint:
        name: Lint
        runs-on: ubuntu-latest
        steps:
          - name: Checkout repository
            uses: actions/checkout@v4

          - name: Setup Node.js
            uses: actions/setup-node@v4
            with:
              node-version: '20'
              cache: 'npm'

          - name: Install dependencies
            run: npm ci

          - name: Run linter
            run: npm run lint

      test:
        name: Test
        runs-on: ubuntu-latest
        needs: lint
        steps:
          - name: Checkout repository
            uses: actions/checkout@v4

          - name: Setup Node.js
            uses: actions/setup-node@v4
            with:
              node-version: '20'
              cache: 'npm'

          - name: Install dependencies
            run: npm ci

          - name: Run unit tests
            run: npm run test -- --coverage=false

      migration-check:
        name: Migration Check
        runs-on: ubuntu-latest
        needs: test
        steps:
          - name: Checkout repository
            uses: actions/checkout@v4

          - name: Setup Node.js
            uses: actions/setup-node@v4
            with:
              node-version: '20'
              cache: 'npm'

          - name: Install dependencies
            run: npm ci

          - name: Check for schema drift
            run: npx drizzle-kit check
    ```

    Key requirements:
    - Name is `CI`
    - Triggers on `pull_request` and `push` to `main` and `develop`
    - Three jobs: `lint`, `test`, `migration-check`
    - Jobs run sequentially (`needs: lint` for test, `needs: test` for migration-check) to save CI minutes if early jobs fail
    - Uses `actions/checkout@v4` and `actions/setup-node@v4` with `node-version: '20'`
    - Uses `npm ci` for deterministic installs
    - `npm run test -- --coverage=false` to skip coverage in CI (faster)
    - `npx drizzle-kit check` for migration validation

    Verify file exists and YAML syntax is valid.
