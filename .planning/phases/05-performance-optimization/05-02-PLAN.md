<objective>
Rewrite the insights service to push all aggregation logic (totals, category breakdowns, daily expenses) to the database via SQL GROUP BY and SUM, eliminating in-memory reduce() loops.

Purpose: As transaction volume grows, fetching all rows into memory and reducing in JavaScript causes OOM kills on Vercel's serverless functions. SQL aggregation is orders of magnitude faster and memory-efficient.
Output: Insights service using pure SQL aggregation with identical API response shape.
</objective>

```typescript
async getMonthlyInsights(userId: string, month?: string) {
  // ... validation ...

  const monthlyTransactions = await this.db
    .select({ amount: transactions.amount, category: transactions.category, createdAt: transactions.createdAt })
    .from(transactions)
    .where(and(eq(transactions.userId, userId), like(transactions.createdAt, `${targetMonth}%`)));

  const total = monthlyTransactions.reduce((sum, tx) => sum + Math.abs(tx.amount), 0);

  const categoryBreakdown = monthlyTransactions.reduce((acc, tx) => {
    const cat = tx.category;
    if (!acc[cat]) acc[cat] = { category: cat, total: 0, count: 0 };
    acc[cat].total += Math.abs(tx.amount);
    acc[cat].count += 1;
    return acc;
  }, {} as Record<string, CategoryBreakdown>);

  const dailyExpenses = monthlyTransactions.reduce((acc, tx) => {
    const date = tx.createdAt.split('T')[0];
    if (!acc[date]) acc[date] = { date, total: 0 };
    acc[date].total += Math.abs(tx.amount);
    return acc;
  }, {} as Record<string, DailyExpense>);

  return {
    month: targetMonth,
    total,
    transactionCount: monthlyTransactions.length,
    categoryBreakdown: Object.values(categoryBreakdown).sort((a, b) => b.total - a.total),
    dailyExpenses: Object.values(dailyExpenses).sort((a, b) => new Date(a.date).getTime() - new Date(b.date).getTime()),
  };
}
```

## Required Output Shape
The response shape must remain identical to avoid breaking the frontend:
```typescript
{
  month: string;
  total: number;
  transactionCount: number;
  categoryBreakdown: CategoryBreakdown[]; // { category, total, count } sorted by total desc
  dailyExpenses: DailyExpense[]; // { date, total } sorted by date asc
}
```

## Schema Note
`transactions.createdAt` is stored as TEXT (ISO 8601 string, e.g. `"2026-04-15T10:00:00.000Z"`). `transactions.amount` is INTEGER (negative for expenses).

## SQL Aggregation Strategy
Use three separate SQL queries:
1. **Total + Count**: `SELECT sum(abs(amount)), count(*) FROM transactions WHERE ...`
2. **Category Breakdown**: `SELECT category, sum(abs(amount)), count(*) FROM transactions WHERE ... GROUP BY category ORDER BY sum(abs(amount)) DESC`
3. **Daily Expenses**: `SELECT date(createdAt), sum(abs(amount)) FROM transactions WHERE ... GROUP BY date(createdAt) ORDER BY date(createdAt) ASC`

In Drizzle ORM, use `sql<number>` for aggregates and `sql<string>` for SQLite `date()` function.
