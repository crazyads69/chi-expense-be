## Vercel Configuration Reference
- `maxDuration`: Maximum execution duration for serverless functions (seconds). Vercel Hobby plan max is 60s, Pro is 300s, Enterprise is 900s. Setting to 30 is safe for all plans.
- `runtime`: Node.js runtime version. Options include `nodejs18.x`, `nodejs20.x`. Must match the application's Node version.

In Vercel v2 config, these fields go inside the build object or as top-level function configuration. For `@vercel/node`, the correct placement is within the build config or using `functions` field:
```json
{
  "functions": {
    "dist/main.js": {
      "maxDuration": 30
    }
  }
}
```

However, the simpler and more common approach for single-function apps is adding to the build object:
```json
{
  "builds": [
    {
      "src": "dist/main.js",
      "use": "@vercel/node",
      "config": {
        "maxDuration": 30,
        "runtime": "nodejs20.x"
      }
    }
  ]
}
```
