<task type="auto">
  <name>Task 2: Initialize Sentry in bootstrap and add exception filter</name>
  <files>src/main.ts</files>
  <action>
1. Import Sentry at the TOP of `src/main.ts` (before dotenv.config() is fine, but after dotenv is safer):
   ```typescript
   import * as Sentry from '@sentry/nestjs';
   ```

2. Add Sentry initialization inside `bootstrap()`, BEFORE `NestFactory.create()`:
   ```typescript
   // Initialize Sentry before app creation
   if (process.env.SENTRY_DSN) {
     Sentry.init({
       dsn: process.env.SENTRY_DSN,
       environment: process.env.NODE_ENV || 'development',
       tracesSampleRate: 0.0, // Error tracking only, no performance tracing (v1.1)
     });
   }
   ```

3. After `const app = await NestFactory.create(...)`, add the Sentry exception filter:
   ```typescript
   // Sentry exception filter for unhandled errors
   if (process.env.SENTRY_DSN) {
     const { SentryExceptionFilter } = await import('@sentry/nestjs/setup');
     app.useGlobalFilters(new SentryExceptionFilter());
   }
   ```
   Note: The import pattern may vary based on @sentry/nestjs version. If `SentryExceptionFilter` is not exported from setup, use the standard approach:
   ```typescript
   import { SentryExceptionFilter } from '@sentry/nestjs';
   app.useGlobalFilters(new SentryExceptionFilter());
   ```

4. Ensure Sentry captures bootstrap-level errors by wrapping the migration check:
   ```typescript
   try {
     console.log('[migration] Checking database migrations...');
     await migrate(db, { migrationsFolder: './drizzle' });
     console.log('[migration] Database migrations up to date.');
   } catch (error) {
     console.error('[migration] Migration failed:', error);
     Sentry.captureException(error); // Capture migration failures
     process.exit(1);
   }
   ```
