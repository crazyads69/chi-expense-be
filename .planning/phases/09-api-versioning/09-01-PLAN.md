<objective>
Introduce URI-based API versioning while maintaining backward compatibility.

Purpose: Enable future breaking changes without breaking existing mobile apps.
Output: Versioned routes alongside existing routes, updated Swagger docs.
</objective>

<tasks>

<task type="auto">
  <name>Task 1: Enable versioning in main.ts</name>
  <files>src/main.ts</files>
  <action>
In `src/main.ts`, add versioning configuration after app creation:

```typescript
import { VersioningType, VERSION_NEUTRAL } from '@nestjs/common';

// After const app = await NestFactory.create(...)
app.enableVersioning({
  type: VersioningType.URI,
  defaultVersion: VERSION_NEUTRAL, // Unversioned routes stay accessible
});
```

Note: VERSION_NEUTRAL means routes without @Version() decorator are accessible at both versioned and unversioned URLs. But we want controllers to explicitly opt-in to version '1'.

Actually, better approach:
```typescript
app.enableVersioning({
  type: VersioningType.URI,
  defaultVersion: '1', // Default version for routes without @Version()
});
```

This way existing routes work at both `/api/...` and `/api/v1/...`.

<task type="auto">
  <name>Task 2: Add @Version('1') to all controllers</name>
  <files>All controller files</files>
  <action>
Add `@Version('1')` decorator to all controllers:

Files to update:
- `src/health.controller.ts`
- `src/transactions/transactions.controller.ts`
- `src/input/input.controller.ts`
- `src/insights/insights.controller.ts`
- `src/categories/categories.controller.ts`
- `src/account/account.controller.ts`

Example for health.controller.ts:
```typescript
import { Version } from '@nestjs/common';

@Controller('api')
@ApiTags('Health')
@Version('1')
export class HealthController { ... }
```

Note: Health controller is at 'api' not 'api/health' because the method has @Get('health'). So the versioned path would be `/api/v1/health`.

Wait, that's wrong. If controller is @Controller('api') and we enable URI versioning, the path becomes `/api/v1` + method path. But health is @Get('health'), so it would be `/api/v1/health`.

Actually, for the health controller, we might want it to be VERSION_NEUTRAL so it works at `/api/health` without version too.

Let's keep it simple: all controllers get @Version('1'), and the defaultVersion in main.ts handles unversioned routes.
