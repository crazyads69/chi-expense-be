Minor items for v1.1 consideration (non-blocking):
1. **Redis health check timeout** — E2E tests show 4s+ timeout when Redis is unreachable; could add a shorter timeout for faster degraded response
2. **Swagger response DTOs** — Could add dedicated response DTO classes for richer OpenAPI schemas (currently using inline schema objects)
3. **Health controller Redis mock** — Unit test mocks Redis at module level; could refactor to inject Redis client for better testability
